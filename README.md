# Risk-Based Quality Engineering in Practice

This project follows the [Practice Software Testing] website as it develops across multiple sprint versions.

The purpose is to build practical skill in risk-based quality engineering by exploring how product risks, testing priorities and regression needs change as the application grows.

## Main Focus

The project focuses on:

- understanding the business purpose behind each feature;
- identifying important product and customer risks;
- finding gaps and ambiguity in requirements;
- using exploratory testing to investigate behaviour;
- deciding what should and should not be automated;
- building reliable Playwright tests around meaningful risks;
- diagnosing whether failures come from the product, test, data, environment or requirement;
- reviewing AI-generated ideas and code critically;
- communicating what the testing proves and what risk remains.

## Application Under Test

The [Practice Software Testing] website provides separate hosted versions for each sprint.

The project will begin with Sprint 1 and progress through later versions as the product evolves.
